# 1、需求总述

## 1.1 需求背景

启航平台是中华财险新一代核心系统的员工视角系统门户，承担「应用的发现与访问」入口职能。当前应用信息分散、更新动态与操作指引缺乏统一承载，员工查找系统、了解变更、获取操作指引的成本较高；同时 PMS（项目管理系统）的发布计划与应用侧的更新动态彼此割裂，发布信息无法自动回流到员工视角的门户。

本期重点打通 **PMS 与启航的双向关联**，以「应用」为中枢，把更新日志、操作手册、应用地图整合到统一的员工门户中，形成「启航—知行—聆听」深化应用工具的雏形。

## 1.2 编写目的

明确本期启航平台在应用管理、更新日志、操作手册、新一代应用地图四个模块的功能范围、业务规则、交互要求与验收标准，作为设计、研发、测试与上线验收的统一依据。

## 1.3 名词释义

| 本需求涉及到的常用名词 | 名词解释 | 边界说明 |
| --- | --- | --- |
| 启航平台 | 中华财险新一代核心系统员工视角系统门户，负责应用的发现、访问与信息聚合 | 本期聚焦员工视角门户侧能力 |
| PMS | 项目管理系统，承载各系统的发布窗口、发布单与项目归属 | 启航仅读取并关联 PMS 数据，不在启航内管理发布流程 |
| PMS 发布窗口 | PMS 中定义的发布时间段（格式：`YYYY-MM-DD` + `常规发布/项目发布`），用于约束发布类更新日志的归属 | 一条发布类更新日志关联一个发布窗口 |
| PMS 系统属主 | PMS 中某系统的责任方标识，用于应用地图按「属主」检索 | 以 tag 形式挂在应用上，可多选 |
| 应用清单域 | 启航侧维护的应用主数据集合（名称、分类、属主、简介等） | 与 PMS 系统通过映射关系关联；数量/分类上线后可配置 |
| 更新日志 | 面向员工的系统变更动态，分「发布类」与「非发布类」两种类型 | 发布类需绑定 PMS 发布窗口与所属 PMS 系统 |
| 操作手册 | 面向员工的应用操作指引，按应用独立维护 | 采用左树右文结构，最多 3 层（2 层目录 + 1 层文档） |
| 新一代应用地图 | 员工/管理人员视角的应用全景视图，支持按名称/属主/分类检索并下钻详情 | 本期新建功能「启航：新一代应用地图」 |
| 应用详情页 | 单个应用的聚合详情页，含「操作手册/更新日志/相关课程/我有建议」四个 Tab | 相关课程对接知行社，我有建议嵌入聆听 |

# 2、产品需求概述

## 2.1 产品蓝图

本期以「应用」为中枢，建立启航与 PMS 的关联，并叠更新日志、操作手册、应用地图三类能力：

- **关联层**：应用管理支持关联 PMS 系统、标记 PMS 系统属主；应用清单域与 PMS 系统建立映射。
- **内容层**：每个应用可绑定更新日志（关联发布单）与操作手册（独立目录树）。
- **展现层**：新一代应用地图提供检索与下钻；应用详情页四 Tab 聚合。
- **开放层**：更新日志、操作手册均提供对外查询接口，供 PMS、应用内等场景消费。

## 2.2 用户角色

| 角色名称 | 角色权限功能 | 备注 |
| --- | --- | --- |
| 系统管理员 | 1. 全部应用的新增、编辑、删除；维护应用清单及 PMS 关联<br>2. 全部应用的更新日志创建、编辑、删除、撤回<br>3. 全部应用的操作手册目录与文档创建、编辑、删除<br>4. 分配角色与数据权限 | 由运营支撑域「角色管理」控制 |
| 应用负责人 | 1. 所负责应用的信息编辑<br>2. 所负责应用的更新日志与操作手册维护 | 由系统管理员授权 |
| 普通员工 | 浏览应用地图、查看应用详情、阅读更新日志与操作手册、提交建议 | 员工视角门户主要用户 |
| 管理人员 | 在员工视角基础上，使用管理人员视角的应用地图与统计/管理入口 | 本期应用地图同时覆盖员工与管理双视角 |

# 3、产品功能需求

## 3.1 应用管理（管理端）— PMS 关联

### 3.1.1 新增/编辑应用弹窗

在现有新增/编辑应用弹窗中新增三个字段，弹窗表单字段完整清单如下：

| 序号 | 字段名 | 控件类型 | 是否必填 | 本期调整说明 |
| --- | --- | --- | --- | --- |
| 1 | 应用名称 | 文本输入 | 是 | 原有字段，最大 20 字符 |
| 2 | 编码 | 文本输入 | 是 | 原有字段，最大 12 字符 |
| 3 | URL地址 | 文本输入 | 是 | 🆕 新增必填字段，placeholder"请输入相对地址或绝对地址"；用于应用详情页「进入系统」跳转 |
| 4 | 关联PMS系统 | 按钮触发 + 搜索下拉面板 | 是 | 🆕 点击按钮展开搜索面板，输入关键字过滤 PMS 系统列表（编码-名称格式），选中后关闭面板 |
| 5 | PMS系统属主 | Tag 选择器 | 是 | 🆕 选中 PMS 系统后展示该系统的可选属主列表；点击候选标签添加、点击已选标签 × 移除；提供「全部填入」「全部清空」快捷操作 |
| 6 | 排序 | 数字输入 | 是 | 原有字段，应用按排序值从小到大排列 |
| 7 | 管理员 | 多选下拉（搜索） | 是 | 原有字段，管理员可管理聆听平台建议、维护版本更新日志 |
| 8 | 应用简介 | 文本域（3行） | 否 | 🆕 新增非必填字段，placeholder"请输入应用简介"，用于应用地图和详情页展示 |

**业务规则：**
- The system shall 在保存应用时校验「URL地址」「关联PMS系统」「PMS系统属主」均非空，缺失则阻止提交并提示。
- When 用户变更「关联PMS系统」选项时，the system shall 自动清空已选属主列表并刷新候选池。
- When 用户未选择PMS系统时，the system shall 在属主区域展示"请先选择关联PMS系统"提示，候选池不可见。
- The system shall 不在应用列表中展示「URL地址」和「应用简介」字段。

### 3.1.2 应用列表

**列表调整：**
- 新增「关联PMS系统」列，展示该应用关联的 PMS 系统名称（未关联显示 `—`）

**筛选区调整：**
- 在现有筛选区下方增加独立一行「关联PMS系统」筛选项，搜索下拉选择（单选），支持输入关键字过滤，数据源同 PMS 系统清单

## 3.2 新一代应用地图（启航）与 应用详情页

### 3.2.1 应用地图

新增「启航：新一代应用地图」功能，提供应用检索与全景视图：

**搜索功能：**
- 提供应用名称搜索框和属主部门筛选下拉，支持同时使用
- 搜索范围覆盖：应用名称、系统分类、属主部门，任一命中即返回

**筛选交互：**
- 命中的应用卡片蓝色边框高亮；未命中的卡片灰度降噪（降低透明度+灰度滤镜）
- 搜索结果以下拉面板展示，hover 可定位到对应卡片位置

**卡片展示：**
- 按业务分类分区排列（前端→中端→后端→工具平台→业务中台→数据中台→监管报送）
- 可点击卡片跳转应用详情页；不可点击卡片（如中台底座/报表类）使用灰色虚线和禁用指针区分

**业务规则：**
- When 用户在搜索框输入关键字时，the system shall 实时过滤应用卡片。
- When 过滤无结果时，the system shall 所有卡片进入灰度降噪状态，搜索下拉面板显示"无匹配结果"。
- The system shall 始终展示全量应用卡片并仅通过视觉降噪区分非命中项，不执行 DOM 移除。

### 3.2.2 应用详情页

从应用地图点击任一应用卡片进入独立详情页，聚合该应用相关信息：

**承载方式：** 独立页面，通过 URL 参数 `?name=应用名称` 加载。

**四 Tab 结构：**

| Tab | 内容 | 本期范围 |
| --- | --- | --- |
| 操作手册 | 展示该应用已发布的操作手册目录与文档内容 | ✅ 完整实现 |
| 更新日志 | 展示该应用已公开的更新日志（按发布时间倒序） | ✅ 完整实现 |
| 相关课程 | 对接知行社，按应用关联培训课程 | Tab 展示，点击后内容区显示「开发中」 |
| 我有建议 | 嵌入聆听平台建议提交功能 | 通过 iframe 或组件嵌入聆听建议表单，建议数据归属聆听 |

**业务规则：**
- When 应用未配置操作手册时，the system shall 在操作手册 Tab 内展示"暂无操作手册"空态。
- When 应用无已公开的更新日志时，the system shall 在更新日志 Tab 内展示"暂无更新日志"空态。
- If URL 参数中的应用名称在应用清单中不存在，then the system shall 展示应用名称作为标题，"该应用的详细信息暂未录入"作为描述。

## 3.3 更新日志

### 3.3.1 管理端：PMS 绑定 + T+1 公开 + 状态调整

#### 3.3.1.1 新增/编辑日志表单

在日志新增/编辑页面的现有表单基础上调整：

**字段清单：**

| 字段名 | 控件类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 标题 | 文本输入 | 是 | 最大 50 字符 |
| 所属应用 | 下拉选择 | 是 | 从应用清单域中选择 |
| 发布类更新 | 复选框 | 否 | 勾选后触发联动（见下方） |
| PMS发布窗口 | 下拉选择 | 条件必填* | 勾选发布类更新时必填；格式 `YYYY-MM-DD 常规发布/项目发布`，数据源来自 PMS |
| 关联PMS系统 | 只读文本 | 条件必填* | 由所属应用自动填入，不可手动编辑；仅发布类更新场景下显示映射值 |
| 日志对外发布时间 | 时间选择 | 条件只读* | 勾选发布类更新时只读并自动填入「发布窗口日期 +1 天 12:00」；未勾选时手动选择 |
| 日志摘要 | 文本域 | 是 | 最大 100 字符 |
| 正文内容 | 富文本编辑器 | 是 | 支持 H1-H3、粗体/斜体/下划线、行内代码、引用块、表格；支持 Markdown 粘贴自动转换 |
| 上传附件 | 文件上传 | 否 | 支持 pptx/xlsx/pdf/docx，可多文件 |

> *条件必填/只读：勾选「发布类更新」时生效。

**发布类更新联动规则：**
- When 管理员勾选「发布类更新」时，the system shall 要求必填 PMS发布窗口，并将关联PMS系统由所属应用自动填入，日志对外发布时间自动计算为「发布窗口日期 +1 天 12:00」且禁止手动编辑。
- When 管理员取消「发布类更新」时，the system shall 恢复所有字段为可编辑状态，清空 PMS发布窗口和关联PMS系统值，日志对外发布时间恢复为手动选择。
- When 管理员变更「PMS发布窗口」选项时，the system shall 重新计算并更新日志对外发布时间。

#### 3.3.1.2 日志查询列表

**状态列：保留原有 Switch 开关作为总控开关。**

| Switch 状态 | 含义 | 对外可见 |
| --- | --- | --- |
| ON | 已公开 | 可见 |
| OFF | 未公开 | 不可见 |

- Switch 开关可由管理员手动操作，作为日志对外可见性的总控兜底。
- 发布类更新日志在「日志对外发布时间」（发布窗口日期 +1 天 12:00）到达时，系统自动将 Switch 置为 ON。
- 管理员可在任意时间手动将 Switch 拨为 OFF 以紧急下架日志（如内容有误需修正）。
- 日志是否已到公开时间通过「发布日期」字段区分，不需要额外增加中间状态。

**列表筛选区调整：**
- 在现有筛选区下方增加独立一行「关联PMS系统」筛选项，搜索下拉选择，数据源同 PMS 系统清单

**其他列表调整：**
- 增加「发布窗口」列（格式 `YYYY-MM-DD 常规发布/项目发布`）
- 列表头统一为「关联PMS系统」

### 3.3.2 对外展示

**应用详情页展示：**
- 更新日志 Tab 展示该应用已公开的日志，按发布时间倒序
- 发布类日志展示「PMS发布窗口」和「关联PMS系统」信息

**对外查询接口：**
- 提供查询接口供 PMS、应用内等场景拉取更新日志
- 入参至少包含：应用标识、状态（已公开）、时间范围
- 仅返回已公开日志，未公开/已撤回日志不可被查询到

## 3.4 操作手册

操作手册是本期全新模块，为每个应用提供独立的结构化操作指引管理能力。面向三类展示端口：普通员工（应用详情页内查看）、管理人员（能力地图查看）、PMS 侧（组件+接口消费）。

### 3.4.1 模块说明

每个应用各自维护一套独立的操作手册目录树，应用间互不干扰。目录结构采用**左树右文**编辑界面，最多支持 **3 层**（2 层目录 + 1 层文档），文档内容通过富文本编辑器编写。

本期不涉及版本控制，操作手册修改后直接覆盖发布。

### 3.4.2 大盘列表

管理端新增「操作手册管理 > 手册大盘清单」页面，汇总展示所有应用的操作手册概况。

**页面布局：** 与更新日志查询页面一致的结构——顶部筛选区 + 工具栏 + 数据表格 + 分页。

**表格列定义：**

| 列名 | 宽度 | 说明 |
| --- | --- | --- |
| 应用编码 | 100px | 等宽字体 |
| 应用名称 | 200px | 加粗展示 |
| 所属大类 | 140px | 应用所属业务分类 |
| 当前操作手册章节数量 | 160px | 该应用已发布的手册文档总数，数字加粗；为 0 时显示「0 篇」 |
| 最后配置维护人 | 140px | 章节数量为 0 时显示 `--` |
| 更新时间 | 180px | 格式 `YYYY-MM-DD HH:mm:ss`，章节数量为 0 时显示 `--` |
| 关联PMS系统 | 130px | 格式「编码-名称」 |
| 操作 | 100px | 「配置」按钮，点击跳转该应用的配置工作台 |

**筛选区：**
- 第一行：应用大类（下拉）、应用名称（文本输入）
- 第二行：关联PMS系统（搜索下拉，独立一行）

### 3.4.3 配置工作台

从大盘列表点击某应用的「配置」按钮进入该应用的专属工作台页面。

#### 页面头部

展示当前编辑的应用名称及编码（如「车险承保平台 [CB001]」），提供「← 返回」按钮回到大盘列表。

#### 编辑器工具栏

### 3.4.4 业务规则

- The system shall 为每个应用维护一套独立的操作手册目录树，应用间互不干扰。
- When 用户在目录树中点击某个文档节点时，the system shall 在右侧编辑器加载该文档的标题和内容。
- When 用户尝试创建超过 3 层的节点时，the system shall 拦截并提示"操作手册最多支持 3 层（2 层目录 + 1 层文档）"。
- When 用户点击「保存」时，the system shall 保存当前编辑内容。若 Switch 为 ON（公开），应用详情页立即可见；若 Switch 为 OFF（不公开），应用详情页隐藏该文档。
- When 用户将 Switch 从 ON 拨为 OFF 时，the system shall 在应用详情页立即隐藏该文档。
- When 用户将 Switch 从 OFF 拨为 ON 时，the system shall 在应用详情页立即展示该文档。
- When 用户粘贴 Markdown 格式文本时，the system shall 自动识别并转换为富文本 HTML。
- If 用户未保存就离开工作台页面时，the system shall 提示"有未保存的更改，确定离开吗？"

### 3.4.5 对外展示

**应用详情页「操作手册」Tab：**
- 展示该应用已发布的操作手册目录树（左侧）和文档内容（右侧）
- 若该应用无已发布的操作手册，展示"暂无操作手册"空态

**对外查询接口：**
- 提供查询接口供 PMS、应用内等场景拉取操作手册
- 入参至少包含：应用标识
- 接口返回目录树结构 + 各文档内容
