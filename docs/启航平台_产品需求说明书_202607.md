# 1、需求总述

## 1.1 需求背景

启航平台是中华财险新一代核心系统的员工视角系统门户，承担「应用的发现与访问」入口职能。当前应用信息分散、更新动态与操作指引缺乏统一承载，员工查找系统、了解变更、获取操作指引的成本较高；同时项目管理（PMS）中的发布计划与应用侧的更新动态彼此割裂，发布信息无法自动回流到员工视角的门户。

本期重点打通 **PMS 与启航的双向关联**，并以「应用」为中心，把更新日志、操作手册、应用地图整合到统一的员工门户中，形成「启航—知行—聆听」深化应用工具的雏形。

## 1.2 编写目的

明确本期启航平台在应用管理、更新日志、操作手册、新一代应用地图四个模块的功能范围、业务规则、交互要求与验收标准，作为设计、研发、测试与上线验收的统一依据。

## 1.3 名词释义

| 本需求涉及到的常用名词 | 名词解释 | 边界说明 |
| --- | --- | --- |
| 启航平台 | 中华财险新一代核心系统员工视角系统门户，负责应用的发现、访问与信息聚合 | 本期聚焦员工视角门户侧能力 |
| PMS | 项目管理系统，承载各系统的发布窗口、发布单与项目归属 | 启航仅读取并关联 PMS 数据，不在启航内管理发布流程 |
| PMS 发布窗口 | PMS 中定义的发布时间段，用于约束发布类更新日志的归属 | 一条发布类更新日志关联一个发布窗口 |
| PMS 系统属主 | PMS 中某系统的责任方标识，用于应用地图按「属主」检索 | 以 tag 形式挂在应用上，可多选 |
| 应用清单域 | 启航侧维护的应用主数据集合（名称、分类、属主等） | 与 PMS 系统通过映射关系关联 |
| 更新日志 | 面向员工的系统变更动态，分「发布类」与「非发布类」 | 发布类需绑定 PMS 发布窗口与所属 PMS 系统 |
| 操作手册 | 面向员工的应用操作指引，按应用独立维护 | 采用左树右文结构，最多 3 层 |
| 新一代应用地图 | 员工/管理人员视角的应用全景视图，支持按名称/属主/分类检索并下钻详情 | 本期新建功能「启航：新一代应用地图」 |
| 应用详情页 | 单个应用的聚合详情页，含操作手册/更新日志/相关课程/我有建议四个 Tab | 相关课程对接知行社，我有建议对接聆听 |

# 2、产品需求概述

## 2.1 产品蓝图

本期以「应用」为中枢，建立启航与 PMS 的关联，并叠更新日志、操作手册、应用地图三类能力：

- **关联层**：应用管理支持关联 PMS 系统、标记 PMS 系统属主；应用清单域与 PMS 系统建立映射。
- **内容层**：每个应用可绑定更新日志（关联发布单）与操作手册（独立目录）。
- **展现层**：新一代应用地图提供检索与下钻；应用详情页四 Tab 聚合操作手册、更新日志、相关课程（知行社）、我有建议（聆听）。
- **开放层**：更新日志、操作手册均提供对外查询接口，供 PMS、应用内等场景消费。

## 2.2 用户角色

| 角色名称 | 角色权限功能 | 所属部门 | 备注 |
| --- | --- | --- | --- |
| 系统管理员 | 全部功能的查看、创建、编辑、删除；维护应用、更新日志、操作手册；分配角色与数据权限 | 创新研发中心 / 业务中台部 | 由角色管理控制 |
| 应用管理员 | 所负责应用的编辑、更新日志与操作手册维护 | 各系统归属部门 | 由系统管理员授权 |
| 普通员工 | 浏览应用地图、查看应用详情、阅读更新日志与操作手册、提交建议 | 全公司 | 员工视角门户主要用户 |
| 管理人员 | 在员工视角基础上，使用管理人员视角的应用地图与统计/管理入口 | 各级管理岗 | 本期应用地图同时覆盖员工与管理双视角 |

# 3、产品功能需求

## 3.1 应用管理（管理端）— PMS 关联

### 3.1.1 新增/编辑应用：关联 PMS 系统 + PMS 系统属主

在新增/编辑应用弹窗中增加两项能力：

- **「关联 PMS 系统」下拉选择**：从 PMS 系统清单中选择本应用对应的 PMS 系统；支持未关联（空）状态。
- **「PMS 系统属主」tag 选择**：以标签形式选择系统属主，可多选；标签来源与 PMS 系统属主一致。

业务规则：

- 当 [event] 管理员在新增/编辑应用弹窗选择「关联 PMS 系统」并保存时，系统应将所选 PMS 系统写入应用清单域的映射关系。
- 当 [state] 应用已关联 PMS 系统时，应用地图与详情页的「所属 PMS 系统」字段应展示该值；当 [unwanted] 关联失败或 PMS 侧该系统不存在时，系统应保留本地选择并提示「PMS 系统映射异常」，避免覆盖已填数据。
- 「PMS 系统属主」为可多选 tag，不影响应用唯一性判定。

验收标准：

1. 新增/编辑应用弹窗可见「关联 PMS 系统」下拉与「PMS 系统属主」tag 选择。
2. 保存后，应用列表与详情页能正确回显关联 PMS 系统与属主 tag。
3. 下拉与 tag 数据源与 PMS 一致；未关联时字段为空且不报错。

### 3.1.2 应用列表：关联 PMS 系统列与搜索

- 列表新增「关联 PMS 系统」列，展示每个应用关联的 PMS 系统名称（未关联显示「—」）。
- 列表搜索/筛选区新增「关联 PMS 系统」搜索项，支持按 PMS 系统名称过滤。

验收标准：

1. 列表默认展示「关联 PMS 系统」列。
2. 使用「关联 PMS 系统」搜索项可过滤出目标应用，结果与列展示一致。

## 3.2 新一代应用地图（启航）

### 3.2.1 应用地图：名称/属主筛选

新增「启航：新一代应用地图」功能，提供应用检索能力：

- **应用名称筛选**：按系统名称模糊匹配。
- **属主筛选**：按 PMS 系统属主过滤。
- 检索结果展示应用卡片，含系统名称、系统分类、属主部门等信息。

业务规则（EARS）：

- 当 [event] 用户在搜索框输入关键字时，系统应同时匹配**系统名称、系统分类、属主部门**三类字段，任一命中即返回该应用。
- 当 [state] 未输入任何关键字时，系统应展示全量应用清单。
- 当 [unwanted] 查询无结果时，系统应展示空状态提示，不报错。

验收标准：

1. 应用地图可打开，提供应用名称与属主两类筛选入口。
2. 输入系统分类或属主部门关键字也能命中对应应用（与名称等效）。
3. 无结果时给出明确空状态。

### 3.2.2 应用详情页：四 Tab

新增应用详情页，包含四个 Tab：

1. **操作手册**：展示该应用关联的操作手册（见 3.4）。
2. **更新日志**：展示该应用的更新日志（见 3.3）。
3. **相关课程**：展示该应用关联的培训课程，对接知行社。
4. **我有建议**：展示/提交该应用的建议与反馈，对接聆听。

业务规则：

- 当 [optional] 应用配置了操作手册时，系统应在详情页显示「操作手册」Tab；当 [optional] 应用未配置操作手册时，系统应隐藏或置空该 Tab 并提示「暂无操作手册」。
- 当 [optional] 应用配置了相关课程/建议通道时，对应 Tab 同理展示；未配置时给出空态。
- 系统应始终（Ubiquitous）在应用详情页展示该应用的更新日志与操作手册入口（若已配置）。

验收标准：

1. 从应用地图点击任一应用可进入详情页。
2. 四个 Tab 均可正常切换，已配置内容正确渲染，未配置内容有空态说明。
3. 相关课程、我有建议分别正确对接知行社、聆听对应页面/接口。

## 3.3 更新日志

### 3.3.1 管理端：PMS 绑定 + T+1 公开 + 富文本

更新日志管理（管理端）本期调整：

- **发布类更新绑定 PMS**：发布类更新需填写「PMS 发布窗口」与「所属 PMS 系统」；非发布类不强制。
- **T+1 自动公开**：发布类更新日志最迟于**发布日第二天中午 12:00** 自动对外公开。
- **富文本编辑器升级**：支持 markdown 格式编辑与渲染。
- **列表筛选**：列表增加「关联 PMS 系统」筛选项。

业务规则（EARS）：

- 当 [event] 管理员将更新日志标记为「发布类」时，系统应强制要求填写「PMS 发布窗口」与「所属 PMS 系统」，缺失则阻止提交。
- 当 [event] 一条发布类更新日志到达「发布日 +1 天 12:00」时，系统应自动将其状态置为「已公开」并向外可见。
- 当 [unwanted] 到达公开时点但 PMS 发布单接口不可用或数据异常时，系统应保留日志为「待公开」并通知管理员，次日同一时点重试，不静默丢弃。
- 当 [state] 更新日志处于「未公开」状态时，系统应对外隐藏其内容（应用详情、对外接口均不可见）。

验收标准：

1. 发布类更新日志必须绑定 PMS 发布窗口与所属 PMS 系统，否则无法提交。
2. 非发布类更新日志可不绑定，流程不受影响。
3. 发布类日志在发布日次日 12:00 自动公开，公开时点可验证。
4. 编辑器支持 markdown 编辑与渲染。
5. 管理端列表支持按「关联 PMS 系统」筛选。

### 3.3.2 对外展示：应用详情 + 查询接口

- 启航应用详情页「更新日志」Tab 展示该应用已公开的更新日志。
- 提供**对外查询接口**，供 PMS、应用内等场景拉取更新日志。

接口要求：

- 入参至少包含「应用标识」「状态（已公开）」「时间范围」。
- 仅返回已公开日志；未公开日志不对外暴露。

验收标准：

1. 应用详情「更新日志」Tab 仅展示已公开内容。
2. 对外接口按应用标识返回已公开日志，未公开内容不可被查询到。

## 3.4 操作手册

### 3.4.1 管理端：独立目录 + 左树右文 + 富文本

操作手册管理（管理端）：

- **每应用独立目录**：每个应用各自维护一套独立的操作手册目录，互不干扰。
- **左树右文界面**：左侧为目录树，右侧为文档编辑/预览区；最多 **3 层**（2 层目录 + 1 层文档）。
- **富文本编辑器**：支持 markdown 格式。

业务规则：

- 当 [event] 应用管理员在其应用下新建手册节点时，系统应将其挂载到该应用的独立目录树下。
- 当 [unwanted] 用户尝试创建超过 3 层的节点时，系统应拦截并提示「操作手册最多支持 3 层（2 层目录 + 1 层文档）」。
- 当 [state] 手册内容含 markdown 标记时，系统应在预览与对外展示中按 markdown 渲染。

验收标准：

1. 各应用操作手册目录相互独立，A 应用改动不影响 B 应用。
2. 目录树最多 3 层，超限被拦截。
3. 左树右文交互正常，markdown 内容正确渲染。

### 3.4.2 对外展示：应用详情 + 查询接口

- 启航应用详情页「操作手册」Tab 展示该应用已发布手册。
- 提供**对外查询接口**，供 PMS、应用内等场景拉取操作手册。

验收标准：

1. 应用详情「操作手册」Tab 正确展示该应用手册目录与文档。
2. 对外接口按应用标识返回手册内容，权限与可见性受控。
