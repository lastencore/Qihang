**内容范围**

本文档是中华财产保险股份有限公司新一代核心系统「启航平台」中「新一代应用地图」页面的产品需求规格说明书，覆盖首屏 Banner、应用全景展示、智能检索、详情跳转等完整功能。

**适用的对象**

本文档仅适用于中华财产保险股份有限公司启航平台项目组以及公司的有关决策者、评估者阅读。

# 1、需求总述
## 1.1 需求背景
启航平台作为中华财险员工视角的系统门户，承载 50+ 应用的发现与访问入口。当前员工查找系统依赖应用列表或自定义收藏，缺乏直观的全景视图和快速定位手段。新增「新一代应用地图」页面，以业务分类分区展示全量应用，配合智能检索和首屏品牌展示，提升员工找应用的效率与体验。

## 1.2 编写目的
明确新一代应用地图页面在首屏展示、应用分区布局、卡片样式、智能检索、详情跳转、滚动交互等方面的功能范围、业务规则、交互要求与验收标准，作为设计、研发、测试与上线验收的统一依据。

## 1.3 名词释义
| 本需求涉及到的常用名词 | 名词解释 | 边界说明 |
| --- | --- | --- |
| 新一代应用地图 | 启航平台客户端页面，以业务分类分区展示全量应用卡片，支持检索与下钻 | 本文档所述全部功能范围 |
| 应用卡片 | 每个应用在地图中的最小展示单元，平铺于所属分类区域中 | 可点击卡片跳转详情页；不可点击卡片仅展示 |
| 不可点击卡片 | 面向中台底座、报表等应用，仅作存在性展示，不跳转详情 | 视觉上以浅灰底色 + 禁用光标区分 |
| 首屏 Banner | 页面打开时展示的全幅品牌区域，含平台标题、副标题与下滚提示 | 占满整个屏幕高度，与地图区域无缝衔接 |
| 边界积累过渡 | 用户从 Banner 下滚到地图（或反向）时的过渡机制：在边界区域滚轮不直接移动页面，而是积累滚动意图，达到一定量后一次性平滑滑入目标区域 | 防止 Banner 与地图之间出现"半截露出"的中间态 |
| 智能检索 | 通过应用名称或属主部门关键字过滤地图中应用卡片的搜索功能 | 搜索范围为名称 + 分类 + 属主，任一命中即返回；搜索结果以下拉面板展示，鼠标悬停可定位 |
| 灰度降噪 | 搜索过滤时，未命中卡片降低透明度和色彩饱和度，从视觉上弱化非匹配项 | 保留全部卡片在原位置，不移除或隐藏 |
| 属主部门 | PMS 中某系统的责任方标识，如「信息技术部」「数据管理部」等 | 对应应用管理端「PMS系统属主」字段 |

# 2、产品需求概述
## 2.1 产品蓝图
新一代应用地图是启航平台员工视角门户的核心入口页面，由三个层级构成：

+ **首屏层**：全幅 Banner 品牌展示，承载平台标识与视觉引导，用户下滚后进入地图。
+ **检索层**：地图区顶部固定搜索框 + 属主筛选下拉，支持名称/属主实时过滤；搜索结果以下拉面板展示，鼠标悬停结果可定位到对应卡片。
+ **地图层**：7 大业务分类分区展示全量应用卡片。其中工具平台与监管报送并列双栏、业务中台与数据中台采用非对称双栏（业务中台卡片较密集、数据中台卡片较大），其余分类按多列网格自适应排列。

## 2.2 用户角色
| 角色名称 | 角色权限功能 | 备注 |
| --- | --- | --- |
| 系统管理员 | 可在管理端维护应用清单、分类与 PMS 关联，地图根据管理端配置自动渲染 | 地图页面仅做展示，不可在地图内编辑 |
| 应用负责人 | 浏览地图、查看所负责应用的详情 | 同普通员工，无额外权限 |
| 普通员工 | 浏览应用地图、使用搜索定位应用、点击卡片进入详情页 | 应用地图的主要用户 |


## 2.3 产品功能需求列表
| 序号 | 业务场景描述 | 场景所需功能 | 产品功能优先级 |
| --- | --- | --- | --- |
| 1 | 员工进入应用地图需要有品牌认知 | 首屏全幅 Banner，展示平台标识，滚轮引导下翻至地图区 | P0 |
| 2 | 员工需要快速找到目标应用 | 顶部搜索框支持名称/属主实时过滤，结果显示下拉面板，鼠标悬停定位到对应卡片 | P0 |
| 3 | 员工需要浏览全部应用的分布 | 7 大业务分类分区展示全量应用卡片，按统一布局排列 | P0 |
| 4 | 部分中台底座/报表类应用暂时不可直达 | 以不可点击卡片样式展示，明示"存在但不可跳转" | P0 |
| 5 | 点击应用卡片进入详情 | 卡片点击跳转应用详情页 | P0 |
| 6 | Banner 与地图之间不出现半截露出 | 滚轮边界积累过渡，积累达标后一次性平滑滑入目标区域 | P1 |
| 7 | 从应用详情页返回地图时跳过 Banner | 详情页返回时直接展示地图区，不再经过 Banner | P1 |

# 3、产品功能需求

## 3.1 首屏 Banner

### 3.1.1 布局与样式
页面打开时首屏展示全幅 Banner，铺满整个屏幕。

**Banner 区构成：**

| 元素 | 预期表现 |
| --- | --- |
| 背景 | 深蓝到浅蓝的渐变背景，全幅铺满 |
| 主标题 | 展示「启航平台」，大字粗体，白色 |
| 副标题 | 展示「新一代应用地图」，中号字，白色半透明 |
| 下滚提示 | Banner 底部中央显示向下箭头图标，向用户提示可下滚进入地图 |

### 3.1.2 滚动行为
用户在 Banner 区域时，向下滚动**不会直接移动页面**，而是先积累用户的滚动意图。当滚动量积累到一定程度后，页面一次性平滑滑入地图区，完成 Banner 到地图的切换。

用户在地图区向上滚动回到 Banner 时同理：积累滚动量达到一定程度后，一次性平滑滑回 Banner 首屏。

### 3.1.3 业务规则
+ Banner 区域滚轮操作不得触发页面内容的位置移动（页面保持显示完整 Banner）。
+ 从 Banner 进入地图和从地图回到 Banner 均通过一次性平滑滚动完成，不允许出现 Banner 和地图各露出一半的中间状态。
+ 用户已进入地图区后，在地图内容内上下浏览不做任何干预，正常自由滚动。
+ 滚动到页面底部后，向上滚动的操作应立即响应，不得有延迟或冻结。

---

## 3.2 智能检索

### 3.2.1 搜索区域
地图区顶部固定一个始终可见的搜索区域，包含两个控件：

| 控件 | 类型 | 预期表现 |
| --- | --- | --- |
| 应用名称搜索框 | 文本输入 | 输入文字后实时触发应用过滤 |
| 属主部门筛选 | 下拉选择 | 列出全部可选属主部门，选中后与搜索框关键字叠加过滤 |

### 3.2.2 搜索匹配规则
搜索范围覆盖以下三个维度，**任一命中即视为匹配**：
+ 应用名称（核心匹配，不区分中英文大小写）
+ 所属分类名称（一级或二级分类）
+ PMS 属主部门

搜索框文字与属主筛选可同时使用，此时需**同时满足**两个条件（名称命中 AND 属主匹配）。

### 3.2.3 搜索结果下拉面板
输入关键字后，在搜索框正下方弹出下拉面板展示匹配结果：

| 属性 | 预期表现 |
| --- | --- |
| 位置 | 搜索框正下方 |
| 列表高度 | 有限高度，结果超出时面板内可滚动 |
| 每条结果 | 左侧展示箭头图标 + 应用名称；右侧灰色小字显示属主部门 |
| 无匹配结果 | 面板中显示「无匹配结果」 |

### 3.2.4 搜索结果悬停定位
鼠标悬停下拉面板中的某条结果时，页面自动滚动至对应的应用卡片位置（将其置于可视区域的中间位置）。此滚动**不得**导致 Banner 露出——如果目标卡片靠近地图顶部，滚动应停留在地图区顶部边界，不能继续向上翻出 Banner。

### 3.2.5 卡片过滤视觉反馈

| 卡片状态 | 视觉表现 |
| --- | --- |
| 匹配命中 | 蓝色边框高亮，从其他卡片中突出显示 |
| 未命中 | 降低透明度并变为灰色调，视觉上退居背景 |
| 无过滤（初始状态） | 正常展示 |

### 3.2.6 业务规则
+ 搜索过滤时，始终在页面中保留全部应用卡片，仅通过视觉强弱区分匹配项与非匹配项，不移除或隐藏任何卡片。
+ 过滤无结果时，全部卡片进入弱化状态，下拉面板显示「无匹配结果」。
+ 清空搜索框或重置属主筛选后，全部卡片恢复初始正常展示状态。
+ 搜索框失去焦点时，下拉面板延迟短暂时间后关闭，确保用户来得及点击结果。
+ 点击搜索结果中的某一项后：清除搜索框内容、恢复全部卡片正常状态、关闭下拉面板，同时页面滚动至目标卡片位置（滚动方式同悬停定位，受地图顶部边界约束）。

## 3.3 应用地图（分类分区与卡片布局）

### 3.3.1 整体布局结构
地图区自上而下排列 7 个一级业务分类分区，每个分区包含该类别下的全部应用卡片。一级分类按以下顺序固定排列：

| 区域 | 一级分类 | 布局模式 | 说明 |
| --- | --- | --- | --- |
| A | 前端系统 | 多列自适应网格 | — |
| B | 中端系统 | 三栏并排（承保 / 核保 / 综合履约） | 每栏内独立排列卡片，不跨栏填充 |
| C | 后端系统 | 多列自适应网格 | — |
| D | 工具平台 + 监管报送 | 双栏并列 | 左栏为工具平台卡片、右栏为监管报送卡片；两栏保持独立标题，高度拉齐 |
| E | 业务中台 + 数据中台 | 左右非对称双栏 | 左栏业务中台卡片数量较多、卡片较小；右栏数据中台卡片数量较少、卡片较大；两栏高度对齐 |

### 3.3.2 一级分类标题
+ 一级分类标题仅展示分类名称，不使用中文编号（如「一、二、…」）。
+ 如有二级分类，二级分类标题保留编号。

### 3.3.3 应用卡片

#### 3.3.3.1 可点击卡片
面向可跳转详情页的应用：

| 属性 | 预期表现 |
| --- | --- |
| 默认样式 | 白色背景，浅灰色细边框，扁平化卡片 |
| 鼠标悬停 | 边框变为蓝色，背景微变亮，鼠标指针变为手型 |
| 点击行为 | 跳转至该应用的应用详情页 |
| 标签 | 部分卡片名称旁附带小型彩色标签（如「重点」「核心」「大屏」） |

#### 3.3.3.2 不可点击卡片
面向中台底座、报表等暂不提供详情页跳转的应用：

| 属性 | 预期表现 |
| --- | --- |
| 默认样式 | 浅灰底色搭配浅灰实线边框，文字颜色略深于底色，保持可读 |
| 鼠标悬停 | 鼠标指针显示为禁用状态（圆圈斜杠），卡片无任何悬停变化 |
| 点击行为 | 点击无任何响应 |
| 视觉意图 | 向用户传达"此应用存在但暂不支持跳转查看"，而非"此应用未上线" |

### 3.3.4 业务规则
+ 应用分类与卡片数据由管理端「应用管理」与「应用分类管理」动态配置，地图页面仅做渲染展示。
+ 应用数量、分类归属及排列均为上线后可配置项，不在地图前端代码中硬编码。
+ 新增大类或子类时，按管理端配置自动在地图中新增对应分区及卡片。
+ 不可点击的应用由管理端标记控制，地图根据标记自动应用对应样式。

## 3.4 辅助导航

### 3.4.1 回到 Banner 的入口
用户进入地图区后，提供两个回到 Banner 首屏的入口：

| 入口 | 位置 | 预期表现 |
| --- | --- | --- |
| 回顶部按钮 | 地图区右下角，始终可见 | 点击后页面平滑滚动至 Banner 首屏 |
| 应用大厅按钮 | 地图区顶部 | 点击后页面平滑滚动至 Banner 首屏 |

### 3.4.2 详情页返回
从应用详情页返回应用地图时，页面应直接展示地图区（跳过 Banner），避免用户在返回后还需再次下滚。

### 3.4.3 业务规则
+ 点击任一回到 Banner 的入口后，应确保后续滚动交互的状态与当前位置一致，不出现滚动跳跃或方向错误。
+ 从详情页返回地图时，地图页面应能识别"从详情返回"这一场景并直接定位到地图区。

## 3.5 字体层级

| 层级 | 字体大小 | 用途 |
| --- | --- | --- |
| 超大标题 | 显著大于正文 | Banner 主标题（「启航平台」） |
| Banner 副标题 | 约正文 1.5 倍 | Banner 模块名称（「新一代应用地图」） |
| 一级分类标题 | 略大于正文 | 各业务分类分区标题 |
| 二级分类标题 / 卡片文字 / 搜索框 | 常规正文 | 子分区标题、卡片内文字、搜索输入与结果 |
| 三级分类标题 | 略小于正文 | 微标题 |

> 实际字号由视觉设计统一规范确定，本表仅定义相对层级关系。

## 3.6 页面适配
+ 页面内容在宽屏下居中展示，有最大内容宽度限制。
+ 中端三栏、业务中台/数据中台双栏等多栏布局在屏幕较窄时自动切换为单列堆叠，确保小屏可正常浏览。
+ 卡片网格列数随可用宽度自适应，在宽屏下尽量维持设计意图排版（如业务中台卡片密集排列、数据中台卡片较大排列）。

---

## 3.7 交互边界

### 3.7.1 滚动约束
+ 在地图区向上滚动时，绝对不允许翻过地图顶部边界露出 Banner 首屏内容。
+ 滚动到页面最底部后，向上滚动应立即响应，不得出现卡顿或暂时无法上滚的情况。

### 3.7.2 搜索联动
+ 搜索过滤时，搜索结果悬停导致的页面滚动不得突破地图区顶部边界（见 §3.2.4）。
+ 点击搜索结果中的某项 → 页面滚动至目标卡片并居中 → 同时清除搜索框内容和过滤状态。

### 3.7.3 详情页返回
+ 从应用详情页返回地图时，直接展示地图区，不经过 Banner。
+ 返回地图后，不自动清除之前的搜索或筛选状态（由用户返回前的地图页面状态决定）。

# 4、与周边系统关系
| 周边系统 | 关系说明 | 数据流向 |
| --- | --- | --- |
| 启航管理端（应用管理） | 应用清单、分类归属、可访问标记、PMS 关联等主数据由管理端维护，地图消费并渲染 | 管理端 → 地图（数据消费） |
| 启航管理端（应用分类管理） | 一级/二级分类的名称与排列顺序由分类管理页面配置 | 管理端 → 地图（数据消费） |
| 应用详情页 | 地图中可点击卡片跳转至对应应用的应用详情页 | 地图 → 详情页（页面跳转） |
| PMS | 属主部门筛选的数据源来自 PMS 系统属主字段 | PMS → 管理端 → 地图（间接数据流） |
