# 启航平台 · 需求说明

> 版本：2026年7月迭代 | PM：陈俊帆 | 原型地址：见索引入口

---

## 一、项目综述

启航平台是中华财险新一代核心系统的员工统一作业门户，承载 53 个用户视角应用的发现、访问、配置与运营管理。本期重点打通 **PMS（项目管理系统）与启航应用的双向关联**，为后续 PMS 发布单自动关联操作手册和更新日志打下数据基础。

**本期核心目标**：PMS 系统关联全链路落地 + 应用详情页独立化 + 更新日志发布流程规范化。

---

## 二、启航应用管理与客户端

### 2.1 PMS 系统关联闭环

**为什么做**：启航应用与 PMS 系统此前互相独立，应用负责人无法在启航侧维护 PMS 归属关系，运营需跨系统手工核对，效率低且易出错。

**做了什么**：

1. **应用管理 → PMS 系统属主（新增）**
   - 新增/编辑应用弹窗增加「PMS系统属主」必填字段，Tag 选择器交互：点击候选标签加入，点击 × 移除，提供「全部填入」「全部清空」一键操作；
   - 候选池为灰色浅底卡片区，标注"可选属主"提示文案；
   - 选择 PMS 系统后自动加载对应属主候选标签。

2. **关联PMS系统下拉重构**
   - 弃用原生 datalist 下拉，改为按钮触发 + 搜索面板模式（参考应用地图风格），支持关键词实时过滤 114 个 PMS 系统。

3. **PMS 筛选能力覆盖**
   - 应用管理、手册大盘、更新日志查询三个列表页均增加「关联PMS系统」筛选项，跨页面统一。

### 2.2 应用详情页独立化

**为什么做**：原有"应用大厅"方案未通过评审，但应用详情（操作手册/更新日志/课程/建议）需作为标准入口保留。

**做了什么**：

1. **新增独立详情页** `app_detail.html`
   - 从应用大厅抽出详情面板，做成独立页面，支持 `?name=系统名` 参数跳转；
   - 四个 Tab：操作手册（左树右文）、更新日志（风琴时间线）、相关课程（网格卡片）、我有建议（聆听平台表单）；
   - 与应用大厅保持相同结构和样式。

2. **应用地图跳转打通**
   - 108 个场景模块卡片点击 → 跳转详情页；
   - 搜索下拉结果悬停定位 + 点击跳转详情页；
   - 属主部门筛选联动结果同样支持跳转。

3. **索引入口整理**
   - `index.html` 入口顺序调整为：应用地图 → 应用大厅（方案待定）→ 应用详情；
   - 各页面描述文案同步更新。

## 三、更新日志

### 3.1 PMS 发布窗口与关联

**为什么做**：更新日志需与 PMS 发布计划对齐，运营需在新增日志时关联 PMS 发布窗口和对应系统。

**做了什么**：

1. **发布类更新联动**
   - 勾选「发布类更新」→ PMS发布窗口（必选）+ 关联PMS系统（自动填入）+ 日志对外发布时间（自动填入次日 12:00，不可编辑）；
   - 未勾选 → 前两个字段非必选，发布时间恢复可编辑。

2. **字段文案规范化**
   - "发布窗口" → "PMS发布窗口"；
   - "发布时间" → "日志对外发布时间"；
   - "应用名称" → "所属应用"；
   - 全部 placeholder 统一为"请输入"/"请选择"。

3. **布局优化**
   - 标题 + 所属应用同行（col-12 两栏）；
   - 发布类更新 checkbox + PMS发布窗口 + 关联PMS系统同行（flex nowrap + 居中对齐）；
   - 日志对外发布时间紧随其后。

4. **导航模板化**
   - 左侧导航和顶栏改为 nav-loader.js 动态加载，与其他管理端页面一致。

### 3.2 更新日志查询增强

- 列表增加「关联PMS系统」筛选项；
- 状态列去除固定定位，日志摘要列缩窄、操作列加宽。
